간단히 비유하자면, 애플리케이션은 **'똑똑한 도서관 키오스크'**이고, 에이전트는 **'나를 대신해 일을 처리해 주는 유능한 비서'**

## 1. LLM 기반 애플리케이션 (LLM-based Apps)
단순히 LLM(거대언어모델)을 활용해 특정 기능을 수행하는 프로그램입니다. 주로 사용자의 입력을 받아 적절한 답변을 생성하는 데 집중합니다.
- **동작 방식**: 사용자가 질문하면(Input), 학습된 데이터를 바탕으로 답변을 생성(Output)합니다.
- **대표적 기술 (RAG)**: 모델이 모르는 최신 정보나 내부 문서를 검색해서 답변에 활용하는 **RAG(검색 증강 생성)** 기술이 주로 쓰입니다.
- **특징**: 수동적입니다. 사용자가 물어봐야 답을 하며, 정해진 프로세스 안에서만 움직입니다.
- **예시**: 챗봇, 문서 요약기, 번역 서비스 등.

## 2. AI 에이전트 (AI Agent)
에이전트는 단순히 답변하는 수준을 넘어, 목표를 달성하기 위해 **스스로 계획을 세우고 도구를 사용하며 실행**하는 시스템입니다.
- **핵심 구성 요소:**
    - **계획(Planning)**: 복잡한 문제를 작은 단위로 쪼개고 순서를 정합니다.
    - **기억(Memory)**: 과거의 대화 내용이나 작업 결과를 저장하고 활용합니다.
    - **도구 사용(Tool Use)**: 필요하다면 계산기를 쓰거나, 구글 검색을 하고, 이메일을 보내는 등 외부 API를 직접 호출합니다.
- **특징**: 능동적입니다. "이번 주 여행 계획 짜고 비행기표 예약해 줘"라고 하면, 검색부터 예약까지의 과정을 판단하여 수행합니다.
- **예시**: 코딩 에이전트(Devin), 자율형 비서 등.

## 3. 한눈에 비교하기
|구분|LLM 애플리케이션 (예: 일반 챗봇)|AI 에이전트 (Agent)|
|---|---|---|
|**주요 역할**|정보 제공 및 텍스트 생성|목표 달성을 위한 작업 수행|
|**자율성**|낮음 (질문에 답만 함)|높음(스스로 판단하고 실행)|
|**상호적용**|일회성 대화 위주|연속적인 루프 및 도구 활용|
|**비유**|지식 백과사전|실제 업무를 수행하는 대리인|
